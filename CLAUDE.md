# TodoWidget - Rainmeter Todo Skin

## Critical Rainmeter Lessons (learned the hard way)

### 1. Double Refresh is mandatory

Rainmeter's `@Include` parses .inc files **before** Lua `Update()` runs. Every action that modifies data needs `[!Refresh] [!Refresh]` (with spaces between bangs):

1. First `[!Refresh]`: triggers Lua `Update()` which writes new DynamicMeters.inc
2. Second `[!Refresh]`: reloads the skin, now picking up the new .inc content

This applies to both `.ini` InputText commands AND Lua-generated meter actions.

### 2. InputText: use `[MeasureInput]`, NOT `$UserInput$`

The only working pattern for InputText is the **two-command batch** using the measure value:

```ini
Command1=["$UserInput$"] X=15 Y=5 W=300 H=28 DefaultValue=""
Command2=[!CommandMeasure "MeasureTodoEngine" "SomeFunc('[MeasureInput]')"] [!Refresh] [!Refresh]
```

**What does NOT work:**
- Single-command with `$UserInput$` directly in the action — the dialog shows but the action never fires
- `$UserInput$` in Command2 instead of `[MeasureInput]` — unreliable, doesn't substitute
- `ExecuteBatch` with a single command number — needs the pair (input + action)

**What DOES work:**
- `["$UserInput$"]` in Command1 to show the dialog and store the value
- `[MeasureInput]` in Command2 to read the stored value
- Called via `[!CommandMeasure MeasureInput "ExecuteBatch 1-2"]`
- Spaces between bangs: `] [!Refresh]` not `][!Refresh]`

### 3. Update=-1 is essential

`Update=1000` (or any positive value) causes the skin to refresh on a timer, which **kills InputText dialogs** before the user can submit. Always use `Update=-1` so the skin only updates on demand.

### 4. DynamicVariables=1 on InputText measure

Required for `#EditIndex#` and `#EditDefault#` variables to resolve at execution time rather than at skin load time. Without it, dynamic variables in commands always resolve to their initial values.

### 5. Lua SplitText: don't use gmatch for pipe-delimited fields

`string.gmatch(str, '([^|]*)')` produces extra empty matches between consecutive delimiters (`||`). This shifts field indices and breaks description parsing. Use a manual split loop with `string.find` instead.

### 6. Font loading

Material Icons font must be installed to the **user fonts folder** (`AppData/Local/Microsoft/Windows/Fonts/`) and registered via registry (`HKCU\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Fonts`). The `LocalFont=` directive in [Rainmeter] alone is not sufficient for fonts used in Lua-generated dynamic meters.

### 7. Rainmeter.ini is UTF-16

The main config at `AppData/Roaming/Rainmeter/Rainmeter.ini` is UTF-16LE encoded. CLI bangs like `!WriteKeyValue` may silently fail to modify it. Edit the binary directly with proper encoding handling if needed.

## Architecture

- `todo/todo.ini` — Main skin config. InputText commands defined here (paired: input + action via `ExecuteBatch N-(N+1)`).
- `@Resources/TodoEngine.lua` — Core logic. `Update()` reads `data.txt` and generates `DynamicMeters.inc`. Action functions (AddSection, InsertTaskAfterSection, etc.) called from skin via `[!CommandMeasure]`.
- `@Resources/DynamicMeters.inc` — Auto-generated by Lua on every Update(). Never edit manually.
- `todo/data.txt` — Plain text data store. `##Label` = section header, `name|checked|description` = task, `>name|checked|description` = subitem.
- `@Resources/Icons.inc` — Material Icons unicode mappings.
